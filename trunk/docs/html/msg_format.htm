Формат сообщений

Описание имеет вид:
число_байт - описание
ХХ - переменное число байт

Размер указывает число байтов, следующих за самим размером. Например, размер сообщения 80 байт, это значит, что сначала идет указатель размера (4 байта), а потом само сообщение (80 байт).

Служебная информация представляет собой набор текстовых строк, разделенных символами перевода строки (CR+LF). Формат данных в строках определяется типом сообщения. Общие правила:

  • Первое слово в строке является ключевым.
  • После ключевого слова может следовать разделитель и параметры (один или несколько).
  • Если в начале строки стоит пробел или таб, значит это продолжение предыдущей строки.
  • Пустая строка (состоящая только из символов перевода строки) означает конец служебной информации.

    Разделителем может быть знак "=" (равно), ":" (двоеточие) или " " (пробел), в зависимости от формата данных. Варианты форматов данных:

  • имя значение1 значение2
  • имя=значение
  • имя: имя1=значение1; имя2=значение2

    Все сообщения имеют формат:

    04 - размер сообщения
    04 - тип сообщения
    04 - таймштамп
    08 - адрес отправителя
    08 - адрес получателя
    04 - смещение от начала до синбаев (размер содержимого + 24 байта)
    04 - размер служебной информации
    XX - служебная информация
    04 - размер содержимого сообщения
    XX - содержимое сообщения
    XX - синбаи (по 4 байта)

    Примеры сообщений

    • AUTH - Аутентификация
    • USRD - Каталог пользователей (Users Directory)
    • SRVD - Каталог сервисов (Services Directory)
    • CHAT - Чат, короткие сообщения между пользователями
    • MAIL - Почта, длинные сообщения между пользователями
    • GRPC - Групповой чат, каналы коротких сообщений
    • FORU - Форумы
    • FLST - Хранилище файлов (Files Storage)
    Сообщения между пользователями

    Сообщения между пользователями

    Приват-чат:
    Тип: CHAT
    Параметры:
    нет
    Данные:
    текст сообщения

    Приват:
    Тип: MAIL
    Параметры:
    source_timestamp - таймштамп исходного сообщения
    header - заголовок
    Данные:
    текст сообщения

    Сообщение на канал:
    Тип: GRPC
    Параметры:
    group - название группы
    Данные:
    текст сообщения

    Сообщение на форум:
    Тип: FORU
    Параметры:
    group - название группы
    header - заголовок
    Данные:
    текст сообщения

    Передача файла получателю

    Передача файла получателю

    Заголовок файла:
    Тип: FILE
    Параметры:
    file_id - ID файла
    file_name - название файла
    file_size - размер файла
    file_time - время файла (UNIX-time)
    segment_size - размер сегмента
    segment_count - число сегментов
    md5 - хеш файла MD5
    Данные:
    содержимое файла (первый сегмент)

    Сегмент файла:
    Тип: FSEG
    Параметры:
    file_id - ID файла
    segment_number - номер сегмента
    segment_size - размер сегмента
    Данные:
    содержимое файла (сегмент)

    Запрос сегмента:
    Тип: SREQ
    Параметры:
    file_id - ID файла
    segment_number - номер сегмента
    Данные:
    нет

    Обмен сведениями об узлах и точках

    Обмен сведениями об узлах и точках

    Для построения карты сегмента можно запросить узел об его линках

    Запрос списка линков (Links Request):
    Тип: LNRQ
    Параметры:
    нет
    Данные:
    нет

    Запрос поинтлиста (Pointlist Request): Тип: PNRQ
    Параметры:
    нет
    Данные:
    нет

    Можно запросить у узла подробную информацию об одном из его поинтов по адресу или части имени.

    Получить информацию об адресе или имени (Get Info):
    Тип: GINF
    Параметры:
    addr - адрес
    name - название
    Данные:
    нет

    Подробная информация возвращается тому, кто ее запросил.

    Информация линка (Link Info):
    Тип: LNKI
    Параметры:
    addr - адрес узла
    name - название узла
    owner - информация о владельце узла
    ip_addr - IP-адрес узла
    phone_no - номер телефона узла
    rating - рейтинг узла
    Данные:
    нет

    Служебные сообщения

    Служебные сообщения

    Сообщение об ошибке (Error Report)
    Тип: ERRR
    Параметры:
    err_code - код ошибки
    err_text - текст сообщения об ошибке
    source_timestamp - таймштамп исходного сообщения
    Данные:
    исходное сообщение целиком

    Информационное сообщение

    Запрос информации, подтверждение доставки сообщения, ответ на запрос, итд..

    Тип: INFO
    Параметры:
    info_type - вид информации в сообщении
    source_timestamp - таймштамп исходного сообщения
    Данные:
    нет
